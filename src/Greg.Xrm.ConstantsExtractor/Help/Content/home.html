Summary

_n.ConstantExtractors can be used to generate C# and JS constants files describing the structure of Dataverse tables and columns.
It can be really useful to simplify work with late bound entities: usage of constants prevents indiscriminated distribution of "magic strings" or "magic numbers" within the code.

Kudos to Gianluca Tocco for the core engine!

Given a specific solution, the tool generates:

  • A constants file for each Table contained in that solution
  • An EntityGenericConstants file, containing the base class for each table constants class
  • A GlobalOptionSetConstants file containing an enum for each global option set available in the environment
  • A GlobalBooleanConstants file containing an enum for each global boolean option set available in the environment

Using the Settings view, you can specify

  • The solution containing the entities put in output
  • If you want to generate C# constants or not
  • The namespace to be used for the C# constants classes
  • The path of the output folder that will contain the generated C# constant classes
  • If you want to generate JS constants or not
  • The namespace to be used for the JS constants objects
  • Optional header lines to be included in the JS constants objects files
  • The path of the output folder that will contain the generated JS constant object files
  • Whether or not to add the column type in the comments of the generated classes/object property comments
  • Whether or not to add the column description in the comments of the generated classes/object property comments

Please note: the tool doesn't prunes the output folder. Removed table classes/objects will remain and they need to be removed manually.
