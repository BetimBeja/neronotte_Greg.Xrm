Result summary

The Result Summary view is automatically filled and displayed when a comparison process completes.

A treeview is displayed showing a node for each compared entity, with 4 subnodes, each one corrisponding to one of the 4 possible cluster of results:

  • Matching: represents a record that is present in both environments, with matching fields. During a deploy, there is nothing to do with such kind of records.
  • Missing on ENV1: represents a record that is present in ENV2 but not in ENV1. During a deploy, probably it should be removed.
  • Missing on ENV2: represents a record that is present in ENV1 but not in ENV2. During a deploy, probably it shoud be added to ENV2.
  • With differences: represents a record that is present in both environments, but has differences in one or more fields.

For each entity, the total number of analyzed records is displayed after the entity name.
In the subnodes, the total number of records of the cluster is displayed after the cluster name, compared with the total number of records in all clusters.

Text color is used to improve the readability of the results:

  • Green is used to identify having all matching records. Those nodes are by default displayed without additional info.
  • Dark gray is used to identify empty clusters.
  • Light gray is used to identify clusters containing at least one record.

Comparison results can be also exported in Excel for further analysis.

Available actions on result clusters

Right-clicking on a node of the Result summary tree view, a context menu shows the list of possible actions that can be applied to the records related to the selected node.
Actions are:

  • Mark as good: Clears the current entity as "good" (green) even if it has differences.
  • Unmark as good: Restores the node status.

Right-clicking on a subnode of the Result summary tree view, a context menu shows the list of possible actions that can be applied to the records related to the selected node.
Actions are:

  • Copy to ENV2: aligns both environments to the values of ENV1. Applicable only on Missing on ENV2 or With differences subnodes.
  • Copy to ENV1: aligns both environments to the values of ENV2. Applicable only on Missing on ENV1 or With differences subnodes.
  • Delete from ENV1: deletes the record from ENV1. Applicable only on Missing on ENV2 or With differences subnodes.
  • Delete from ENV2: deletes the record from ENV2. Applicable only on Missing on ENV1 or With differences subnodes.
