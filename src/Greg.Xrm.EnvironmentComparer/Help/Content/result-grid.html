Comparison result details

The comparison result details grid shows the result of the comparison for a specific entity.

You can pick the result set to be displayed in the comparison result details grid from the Result Summary view.

Result conventions

The color of each line of the grid matches a specific type of result

  • Green: represents a record that is present in both environments, with matching fields. During a deploy, there is nothing to do with such kind of records.
  • Orange: represents a record that is present in ENV2 but not in ENV1. During a deploy, probably it should be removed.
  • Azure: represents a record that is present in ENV1 but not in ENV2. During a deploy, probably it shoud be added to ENV2.
  • Red: represents a record that is present in both environments, but has differences in one or more fields.

Grid columns

  • Key: contains the comparison key for the record.
  • Result: a string representation of the comparison result
  • Columns: in case of a record that is present in both environment with different values, contains the list of columns containing "mismatching values", separated by pipe |.
  • ENV1 values: in case of a record that is present in both environment with different values, contains the list of the "mismatching values" from ENV1, separated by pipe |.
  • ENV2 values: in case of a record that is present in both environment with different values, contains the list of the "mismatching values" from ENV1, separated by pipe |.

Available actions on result records

Right-clicking on one or more rows of the comparison result details grid, a context menu shows the list of possible actions that can be applied to the selected records.
Actions are:

  • Copy to ENV2: aligns both environments to the values of ENV1. Applicable only on Azure and Red rows.
  • Copy to ENV1: aligns both environments to the values of ENV2. Applicable only on Orange and Red rows.
  • Delete from ENV1: deletes the record from ENV1. Applicable only on Azure and Red rows.
  • Delete from ENV2: deletes the record from ENV2. Applicable only on Orange and Red rows.
  • Compare: opens the Record info panel that allows to easily compare each single mismatching attribute value. Applicable only if the user selected one single record.

You cannot apply more than one action for each record.
When an action has been defined for a specific record, the record background is set to Grey, and the action description is shown in the Actions panel.

When selecting more than one record, actions are available only if applicable for all the selected records.
